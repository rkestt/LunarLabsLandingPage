description: Log development progress to DEVLOG.md
---

Log development progress to DEVLOG.md automatically.

**Steps:**

1. Read the current DEVLOG.md
2. Add new entry with:
   - Date (YYYY-MM-DD)
   - What was done
   - Files changed
3. Write updated DEVLOG.md
4. Show summary of what was logged

**Input**: Description of what was completed (or use "auto" to auto-detect from recent git commits)

---

**Example usage:**
- `/dev Added Docker configuration`
- `/dev Implemented user authentication`
- `/dev auto`
